# Custom Archiver FileNet Destination
Author - andy.little@oracle.com 

## Installation
1. Deploy JAR file to `docfactory\lib` on all nodes.
2. Deploy `target\lib` files to `docfactory\lib` on all nodes. Note: if a shaded JAR is used, this step can be skipped.
3. Use a SQL client to execute the following statement. This will enable the use of the custom destination. Note that the `DISPLAY` and `DESCRIPTION` values may be changed, but `ID` (`FileNet`) must remain as-is since it correlates to other settings below.
```sql
INSERT INTO DMKR_ADMIN_1270.DMKR_TRANSLAT (SYS_ID, AL_ID, APP_ID, LOCALE_ID, GROUP_ID, ID, DISPLAY, DESCRIPTION, DISPLAYSEQ) VALUES (1, 1, 0, 'en', 'BCHINGS.BCHINGARCDEST',  'FileNet', 'FileNet', 'Swagger API for FileNet Integration', 1);
```
